RECONSTRUCTION D'UN INDEX ENDOMMAGE

Il peut arriver qu'on ait des incohérences dans certains fichiers du fait que des index ont ete endommagés par un incident de fonctionnement de l'ordinateur. Si le problème n'est pas trop grave, on peut essayer de reconstruire l'index considéré.

Si cette operation de suffit pas, il faudra utiliser des procédures plus lourdes par l'intermediaire du 4D Util.

Allez dans Paramètres, utilitaires,sélectionnez l'option Reconstruction d'un index. Si vous navez pas cette option dans les menus, allez en Paramètres, Utilitaires, Executer une procedure, et demandez l'execution de la procedure Reindexer.

Supposons que vous désiriez reconstruire l'index du champ NumeroCpte de la table Journaux. Dans la zone de défilement de gauche, vous avez la liste des tables existant dans Assur.

Vous cliquez sur le nom de la table concernée, dans notre exemple Journaux.

La liste des champs de la table Journaux s'affiche dans la zone de défilement de droite.

Selectionnez le champ NumeroCpte.

Cliquez sur le bouton Désindexer.

Une alerte vous indique que l'index a été supprimé.

Cliquez sur OK.

Puis Cliquez sur Reindexer.

;Une alerte vous previent lorsque la reindexation es terminée. Cliquez sur OK, puis sur Validation

Allez voir si votre probleme persiste. Si c'est le cas il faudra utiliser Le programme de réparation 4D Util.
