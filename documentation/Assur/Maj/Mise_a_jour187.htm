MODIFICATIONS ASSUR   MAJ 187 AU 23/7/2002

 

Page précédente

 

1) Evolutions

Pour les imports et exports de données :jusqu'à maintenant, le répertoire (ou dossier)  de travail était imposé : Répertoire c:\Assur en monoposte, répertoire c:\ACI\Programs\4DCLI606 en multiposte

Vous pouvez désormais désigner le répertoire de travail par défaut dans lequel vous placerez vos documents dont le contenu doit être importé.

Vous pouvez le faire globalement pour tous les utilisateurs : allez en paramètres généraux, cliquez sur le bouton Accès et inscrivez dans la rubrique Répertoire de travail l'adresse de ce répertoire. Exemples : c:\MPU\ , ou c:\AXA\ ou c:\Imports\ ......

Vous pouvez aussi affecter à chaque utilisateur un répertoire de travail.

Pour cela, allez en gestion des collaborateurs, cliquez sur le bouton Personnaliser et inscrivez dans la rubrique Répertoire de travail l'adresse de son dossier de travail.

Deux avantages à ce système : Vous n'aurez plus à déplacer les fichiers transmis par les compagnies, il suffira de désigner comme répertoire de travail celui dans lequel les fichiers arrivent . Par ailleurs, des popu_menus vous permettront de sélectionner le fichier sur lequel vous désirez travailler, sans avoir à saisir son nom exact.

 

Pour les Requêtes Recherches et Etats :  il n'était pas possible jusqu'à maintenant d'accéder sans programmation aux informations de la table de niveau supérieur. Exemple : sur un état Contrats, on ne pouvait pas simplement accéder au nom, ou à la commune du client. Désormais, cela est possible : vous pouvez dans un état Contrats faire imprimer le nom du client, et faire un tri sur le code postal par exemple. De même, sur un état sinistres, vous accédez automatiquement aux informations du contrat et du client etc.......
2) Correction de problèmes signalés :

Cartes vertes : Pour ceux qui utilisent un format de carte verte préparamétré dans Assur, il y avait un problème lorsqu'on demandait l'impression pour une flotte. Dans ce cas, le dialogue d'impression était présenté avant l'impression de chaque carte, ce qui pouvait représenter beaucoup de clics !

Le dialogue n'est désormais présenté qu'une seule fois.
