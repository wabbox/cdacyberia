CONFIGURATION MATERIELLE
La version multipostes de e_assur fonctionne en architecture   Client - Serveur.
Lors de l'installation du logiciel, le réseau doit être configuré et opérationnel.

Nous conseillons vivement d'avoir un serveur dédié, et autant d'ordinateurs qu'il y a d'utilisateurs.

Nous vous rappelons les contraintes minimum matérielles pour une configuration 4 postes : Le serveur dédié et 4 postes clients soit 5 ordinateurs.

Le Serveur dédié : Processeur INTEL - 4 Go de RAM - Disque 60 Go minimum - Graveur de CD ROM  ou graveur DVD - Windows 2003 ou 2008 serveur    Small Business Serveur  OSX Lion Server- Carte Réseau. et un système de sauvegarde permettant à l'utilisateur de vérifier la validité de ses sauvegardes.

Le poste Client : Processeur INTEL -  4 Go de RAM - Disque60 Go minimum (et plus en fonction des autres logiciels installés sur l'ordinateur). -  Lecteur CD ROM 
Windows   XP  ou 7 Mac OSX Lion- carte réseau

NB : les capacités des disques sont à augmenter si vous utilisez la G. E.D : gestion electronique de documents. (stockage des images, des documents scannés.....)

Pour une version 4 postes ou plus, nous conseillons vivement le serveur dédié sous  WIndows Serveur 2003 ou 2008  Small Business Serveur, Mac OSX Lion Server

Nous comprenons toutefois qu'il est difficile pour une configuration de 2 ou 3 postes d'acquérir un ordinateur supplémentaire.

Mais il faut bien comprendre que dans ce cas, on doit installer sur une des machines :

- à la fois le Serveur de Données et le Logiciel Client.

Le fonctionnement de ce poste de travail sera  donc  ralenti.

Nous vous conseillons pour  ce poste qui sera à la fois  Serveur et Client :

8 Go de RAM ou plus
Disque : Capacité à votre convenance de 60 Go minimum - graveur de CD ROM ou DVD
Si vous utilisez des logiciels de Gestion Electronique de Documents (GED), pensez au stockage des documents scannés (augmenter la capacité du disque dur). 

Les mises à jour du logiciel se faisant via Internet, vous devez avoir une connexion et connaître le principe du téléchargement. Votre fournisseur de matériel aura installé tous les éléments nécessaires à votre connexion en fonction de vos configurations (routeurs.......;)

ATTENTION : CDA NE PREND PAS EN CHARGE LES PROBLEMES DE CONNEXION INTERNET SI NOUS N'AVONS PAS LIVRE, INSTALLE ET PARAMETRE VOTRE EQUIPEMENT.

Vous trouverez, ci-dessous, des instructions à communiquer à votre fournisseur de matériel.

INSTALLATION DU RESEAU PAR VOTRE FOURNISSEUR DE MATERIEL

L'installation de notre logiciel suppose que certains paramètres soient configurés.

Ci-dessous, la liste des éléments que doit installer votre fournisseur, afin que la mise en place de notre logiciel, se fasse dans des conditions optimales.

MERCI DE LUI TRANSMETTRE CES INFORMATIONS LORS DE VOTRE INSTALLATION

1°) PARAMETRER L'AFFICHAGE DES OPTIONS

Dans la configuration de l'affichage, les éléments suivants doivent être paramétrés

- Afficher tous les fichiers  - Ne pas masquer les extensions des fichiers  - Afficher le chemin d'accès complet dans la barre de titre.

Ces 3 éléments, nous permettent, lors de la maintenance téléphonique, d'avoir tous les éléments pour analyser tel ou tel type de fichier.

2°) PROTOCOLE RESEAU

Le fournisseur doit installer les protocoles

- TCP / IP 

Si le serveur est un serveur DHCP, donner une durée très longue au bail de chaque client.

3°) VOISINAGE RESEAU

Le disque de l'ordinateur serveur doit être partagé afin que vous puissiez avoir accès, si nécessaire via un poste client, à des informations sur le disque serveur où sera installé le logiciel.

SI POSSIBLE donner comme nom à l'ordinateur serveur SERVEUR.

Si ce n'est pas possible, vous devez connaître le nom du serveur.

Vous devez connaître également le nom de chaque disque "utilisateur".

4°) TEST RESEAU

Les tests effectués par votre fournisseur doivent porter non seulement sur le fonctionnement du Netbeui (voisinage réseau) mais aussi sur le bon fonctionnement du TCP / IP. : que tous les ping sur les postes en réseau soient positifs.

Il faut également tester les imprimantes à partir de chaque ordinateur.

 

Les tests doivent avoir également été faits pour tester le lecteur de sauvegarde.

L'utilisateur doit donc connaître le principe de la sauvegarde.

Attention : un pare-feu mal configuré empêche e_assur de fonctionner.

Les postes en réseau local doivent être acceptés par le pare-feu.

5°) IL EST IMPERATIF QUE LES REPERTOIRES 4D  ET e_Assur qui correspondent à notre applicatif  SOIENT SUR LE DISQUE C A LA RACINE

Pour les PC avec un répertoire WinNT ou Windows ce répertoire doit être accessible par les utilisateurs en lecture et en écriture (accès complet).

Les difficultés d'utilisation dues au partitionnement des disques ne seront pas gérées par l'assistance CDA.

6°) SI VOTRE FOURNISSEUR INSTALLE UN ROUTEUR :

Celui-ci doit impérativement vous transmettre les codes d'accès au routeur ainsi que son adresse IP (privée et publique)

Les codes d'accès se composent d'un nom et d'un MOT DE PASSE.

Si vous désirez travailler a distance, le port 3389 doit être ouvert.

Si vous désirez utiliser la sauvegarde sur Internet, les ports 20 et 21 doivent être ouverts.

5°) ACCES PAR L'UTILISATEUR AU SERVEUR DEDIE

L'utilisateur doit pouvoir accéder au Serveur dédié. Il doit donc connaître les mots de passe nécessaires pour l'accès à l'ordinateur.
